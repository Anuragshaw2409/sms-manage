# Project Structure
This document maps the physical layout of the codebase.

## Directory Layout
```text
sms-management/
├── app/                  # Next.js App Router
│   ├── (protected)/      # Authenticated routes (dashboard, plans, taxes, transactions, users)
│   ├── api/              # API Route Handlers
│   ├── login/            # Public authentication routes
│   └── layout.tsx        # Root layout definition
├── components/           # Reusable React components
│   ├── ui/               # shadcn/ui primitive components
│   └── ...               # App-specific components (sidebar, nav, tables)
├── hooks/                # Custom React hooks
├── lib/                  # Utilities, Prisma instantiations, configuration helpers
├── prisma/               # Prisma schema and configuration
└── public/               # Static assets
```

## Key Files
- `package.json`: Project dependencies and scripts.
- `migration.sql`: Raw SQL migrations generated by Prisma.
- `prisma.config.ts`: Configuration for Prisma.
- `next.config.ts`: Next.js behavior configuration.
- `eslint.config.mjs`: Central linting rules.

## Conventions
- **Colocation**: Routes are defined by folders in `app/` containing `page.tsx` or `layout.tsx`.
- **Component Isolation**: The `components/ui/` folder strictly contains highly reusable UI primitives, while feature-specific complex components (e.g., `data-table.tsx`, `chart-area-interactive.tsx`) sit at the root of `components/`.
